§ 01 — TABLE OF CONTENTS

八件事,
一份目录。

按"输入 → 能力 → 结构"三层组织。每条带定义、典型用法、最小可跑代码,以及最容易踩的坑。

I · 01 Input Layer · 01Prompt 模型读到的全部输入 —— 指令、上下文、示例、约束的总和。是和模型对话的"程序"。 → 第 一 章 ↗ I · 02 Input Layer · 02Context 模型一次能"看见"的全部信息。窗口有限,怎么塞、塞什么、什么时候清掉,是个工程问题。 → 第 二 章 ↗ I · 03 Input Layer · 03Memory 跨会话保留的信息。短期在 context 里,长期写在外面再按需取回 —— 三种典型形态。 → 第 三 章 ↗ II · 04 Capability · 04Tool 让模型能"动手"的东西。搜索、查询数据库、发邮件 —— 模型通过调用 Tool 真正改变外部世界。 → 第 四 章 ↗ II · 05 Capability · 05Skill 可复用的 prompt + tool + 文档的组合单元。把"做一件事"打包成一次调用,而不是每次重写。 → 第 五 章 ↗ II · 06 Capability · 06Agent 能自主规划、调用工具、达成目标的程序。一个 prompt + 一组 tool + 一个 loop,基本就够了。 → 第 六 章 ↗ III · 07 Structure · 07Loop Agent 反复执行"观察 → 思考 → 行动"直到目标达成的循环结构。ReAct 是最经典的一种。 → 第 七 章 ↗ III · 08 Structure · 08Topology 多个 Agent 怎么连。串行、并行、监督 —— 不同的拓扑对应不同的组织哲学,各有取舍。 → 第 八 章 ↗
§ 02 — THE THREE LAYERS

三个层次,
一种切分。

这些概念大致落在三层上 —— 越往上越靠近"做事",越往下越靠近"模型本身"。

A · INPUT LAYER

输入层

你给模型什么。这一层处理"对话",决定了模型对任务的理解深度与稳定性。

  • Prompt01
  • Context02
  • Memory03
B · CAPABILITY

能力层

模型能做什么。这一层把对话变成"做事",是 Agent 真正落地的地方。

C · STRUCTURE

结构层

这些能力怎么组合。Loop 是单体的节奏,Topology 是多体的编排。

B · 04 / 05 / 06

Tool · Skill · Agent

从原子动作(函数),到可复用组合(技能),再到自主目标(代理) —— 三级跃迁。

  • Skill05
  • Agent06
C · 07 / 08

Loop · Topology

单体的循环(ReAct、Reflection),多体的编排(串行、并行、监督) —— 节奏与拓扑。

  • Loop07
  • Topology08
