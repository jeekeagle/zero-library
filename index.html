核心概念

Agent 工程里反复出现的 8 个关键词 —— 每条带一个简短解释和一个代码片段。

查看全部 →
01 / CORE

Prompt

模型读到的全部输入。包括指令、上下文、示例、约束 —— 是和模型对话的"程序"。

基础 文本
02 / CORE

Context

模型一次能"看见"的全部信息。窗口有限,所以怎么塞、塞什么、什么时候清掉,是个工程问题。

窗口 工程
03 / CORE

Tool

让模型能"动手"的东西。搜索、查询数据库、发邮件 —— 模型通过调用 Tool 真正改变外部世界。

函数 动作
04 / CORE

Memory

跨会话保留的信息。短期记忆是当前 context,长期记忆是写在外面再按需取回的东西。

短期 长期
05 / CORE

Skill

可复用的 prompt + tool + 文档的组合单元。把"做一件事"打包成一次调用,而不是每次重写 prompt。

打包 复用
06 / CORE

Loop

Agent 反复执行"观察 → 思考 → 行动"直到目标达成的循环结构。ReAct 是最经典的一种。

循环 ReAct
07 / CORE

Topology

多个 Agent 怎么连。串行、并行、监督 —— 不同的拓扑对应不同的组织哲学,各有取舍。

串行 并行
08 / CORE

Agent

能自主规划、调用工具、达成目标的程序。一个 prompt + 一组 tool + 一个 loop,基本就够了。

自主 目标

写在前面

为什么要把"概念"单独整理出来。

Agent 时代最大的变化,不是模型变强了 —— 是问题被重新切分了。Prompt、Context、Tool、Memory…… 这些词以前只在论文里出现,现在每个工程师每天都要处理。

整理一份关于它们的"小词典",不是为了背下来,而是为了在混乱的术语里,能找到一个共同的支点。

—— Zero-Library · 开篇

三个层次

这些概念大致落在三个层次上 —— 越往上越靠近"做事",越往下越靠近"模型本身"。

输入层 · Input

你给模型什么。Prompt、Context、Memory。

能力层 · Capability

模型能做什么。Tool、Skill、Agent。

结构层 · Structure

这些能力怎么组合。Loop、Topology。
