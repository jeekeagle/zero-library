为什么要整理这些概念

Agent 时代的关键词越来越多,先把最常用、最重要的一组钉住。

一问题的提出

2024 年开始,做 AI 工程的人和写 Prompt 的人开始用同一套术语。

但术语太多、更新太快 —— 一会儿 function calling,一会儿 tool use;一会儿 RAG,一会儿 context window;一会儿 agent,一会儿 workflow。

新人来学,老兵自己讲,讲的都不是同一个东西。这本小书,想做的就是:把这一代 AI 工程里反复出现的、最重要的 8 个概念,用尽量短的篇幅讲清楚。

二这本书长什么样

三编八章,加一个序言和一个索引。

  • 第一编 · 输入层 —— 你给模型什么。Prompt、Context、Memory。
  • 第二编 · 能力层 —— 模型能做什么。Tool、Skill、Agent。
  • 第三编 · 结构层 —— 这些能力怎么组合。Loop、Topology。

每章按"定义 → 为什么重要 → 典型用法 → 一个最小代码片段 → 常见误区"的固定结构写。你可以直接跳到感兴趣的那一章,不必从头读起。

三几条原则

写之前先告诉你我会怎么写、不会怎么写 ——

  • 不追新词。 这一章如果下周有新术语替换,我宁可改成"待补充",也不要把没想清楚的词塞进来。
  • 带最小例子。 每个概念至少有一个 5 行以内能看懂的代码片段。
  • 承认边界。 我不知道的会写"不知道",而不是装作知道。
  • 开源协作。 任何人都能在 GitHub 上提 PR 加章节、改错别字、补充反例。

四它不是什么

  • 不是教程 —— 不会一步一步教你搭 Agent。
  • 不是论文 —— 不会做引用、做对比实验。
  • 不是导购 —— 不会告诉你"哪个框架最好用"。

它更像你写代码时随手开的一个 concepts.md,只不过这个 concepts 放在 GitHub Pages 上,谁都能看、能改、能加。

如果你第一次来,建议先读 Prompt 和 Agent —— 一头一尾,把整本书的脊梁搭起来。
