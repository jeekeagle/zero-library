关于这间小书铺

关于它的来历、原则与未来。

Zero-Library 是一个关于 Agent 时代核心概念的小型开源站点 —— 这里陈列的不是教程,也不是论文,是一组反复出现的关键词 + 最短的解释 + 最小可跑的代码。

这里收什么

两类东西,大致按这个顺序:

  • 概念 (Concept) —— 一个具体的词,比如 Prompt、Context、Tool、Agent。它有自己的定义、自己的边界、自己常见的使用场景。
  • 关系 (Relation) —— 这些概念之间怎么连,谁包含谁,谁和谁配合。结构层(Loop、Topology)主要干这件事。

几个原则

  • 极简。 每章尽量控制在 5 分钟能读完的体量。
  • 带最小例子。 没有代码的概念不写,因为它在 AI 工程里没有意义。
  • 承认边界。 我不知道的会写"未知",不装懂。
  • 开源协作。 任何人都能在 GitHub 上提 issue / PR,补充新概念、修改错别字、给出反例。
  • 没有评论、没有订阅、没有广告。 RSS 是给愿意订阅的人留的,其余请自便。

它不是什么

  • 不是 《百种 AI 概念大全》(系统性的东西会被快速打脸)。
  • 不是教程站(我写的时候读者是未来的我自己,不是小白)。
  • 不是 newsletter(我受不了每周交稿的压力)。

它更像你写代码时随手开的一个 concepts.md,只不过这个 concepts 放在 GitHub Pages 上,谁都能读。

以后会怎样

不知道。这本身就是一种实验 —— 在概念爆炸的年代,坚持把"基础款"讲清楚,有没有价值。

我的赌注是:有,但要慢、要真、要少。

如果你也想整理自己的概念词典,无论公开不公开,我都建议开一个这样的站。不是为了涨粉,是为了让明天的自己,能回看到今天以为"懂了"的那些词,到底懂了几分。
